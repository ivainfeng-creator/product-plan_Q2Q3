VIVERSE · V1.3 Detail
Campaign Evolution
V1.3 · Campaign detail
Campaign evolution paths
Why different campaign types
V1.2's "complete all tasks" model is a baseline. V1.3 tests which structure drives the least drop-off and best completion rate.
🟡All-or-nothing design has the highest drop-off risk.
🟡Multi-entry: more tasks = more chances = less abandonment mid-way.
🟡Tier model gives partial reward — no one leaves empty-handed.
🟡Stamp type feels like discovery, not a homework checklist.
Campaign types to build and test
Completion type — V1.2 baseline
Do all tasks → earn draw entry. Simple and clear. High drop-off risk — all or nothing. This is the benchmark to beat.
V1.2 modelbaseline
Multi-entry type — V1.3 recommended test
More tasks = more draw entries. Login = 1 entry, play 3 games = +1, watch video = +1. Reduces all-or-nothing abandonment.
Test in V1.3lower drop-off
Tier / Milestone type
Bronze (2 tasks) / Silver (4 tasks) / Gold + draw (all tasks). No one leaves empty-handed. Reduces mid-funnel drop-off.
Test in V1.4
Stamp / Explore type
Try 1 game + Watch 1 video + Visit 1 World = collect all stamps → draw entry. Feels like platform discovery, not a task list.
Test in V1.4content discovery
New user vs Return user split
New: login + 1 game + 1 video. Return: 3 logins + 3 games. Different task difficulty, same campaign shell. Better conversion for both segments.
V1.4 prioritysegmentation
Notes
Don't build all types at once. V1.3 tests Multi-entry against V1.2 baseline. V1.4 adds Stamp / Explore and segmentation. Key question: which structure has lowest drop-off and highest completion rate?

V1.3 · Widget detail
Campaign Widget — home screen re-entry
Why widget over push
Push notifications get ignored, turned off, or blocked over time. Widget stays on the home screen permanently — users see it every time they unlock their phone, with zero friction.
🟡Passive visibility — no notification fatigue, no opt-out risk.
🟡Campaign progress visible without opening the app.
🟡1-tap entry direct to campaign landing — no extra navigation.
🟡US users are highly familiar with widgets on both iOS + Android.
Widget spec — App Team
Campaign progress display
Shows current campaign name + X/Y tasks completed. Progress bar updates in real-time. Both iOS (WidgetKit) and Android (App Widget) supported.
P0iOS + Android
1-tap deep link to campaign
Tapping widget routes directly to campaign landing page with correct campaign context. No extra steps or home screen navigation.
P0Deep link
Deadline reminder state
Widget shows urgency indicator when campaign ends in <48h. Visual state change only — not a push. Respects user attention without interrupting.
campaign-aware
Draw entry status
When user qualifies for draw: widget shows "You're entered!" state. Positive reinforcement visible on home screen every day.
reward signal
Widget add flow (in-app CTA)
After joining campaign: prompt to add widget with step-by-step guide for iOS and Android. Shown once, not repeated. Maximizes widget adoption rate.
onboarding
Notes
This widget is campaign-specific — tied to the running campaign, only valuable when a campaign is live. The general mission widget (always-on, not campaign-dependent) comes in V1.5. Don't conflate the two different widgets.
